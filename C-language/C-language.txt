Q.. Pointer arthimatic?
==> pointer arithmatic allow you to perform arithmatic operation on pointer 
	arithmatic operations like increment,decrement,addition,substraction etc.
	
	1. increment
		increment operator increase the pointer to the next memory location of its datatype.
		int a[]={10,20,30,40};
		int *ptr=a;
		ptr++;
		printf("%d\n",*ptr);//output:20;
		
	2. Decrement
		decrement operator decrease the pointer to previous memory location.
		int a[]={10,20,30,40};
		int *ptr=a+1;
		ptr--;
		printf("%d\n",*ptr);//output 10
		
	3. Addition
		addition of a pointer moves pointer forward by n elements based on the size of data type.
		e.g
		int a[]={10,20,30,40};
		int *ptr=a;
		ptr=ptr+2;
		printf("%d\n",*ptr);//output 30
		
	4. Substraction
		substraction of a pointer moves pointer previous by n elements based on size of data type.
		e.g
		int a[]={10,20,30,40};
		int *ptr=a+3;
		ptr=ptr-1;
		printf("%d\n",*ptr);//output 30

Q.. Dynamic memory allocation?
==> - dynamic memory allocation means allocating memory at runtime rather than at compile time.
	- this give flexibility in managing memory for data structure whose size 
	  is known in advance,such as arrays,list or matrics
	- C-Language provide several functions from the <stdlib.h> to handle dynamic memory allocation and deallocation.
	- these function allow use to request memory,resize it and free it when we are done.
	
	Below are the main function of memory allocation
	
	1. malloc()
		- malloc() stand for memory allocation
		- It allocate a block of memory of a specific size.
		- Returns a pointer to the allocated memory.
		- If the allocation is fail then it return null.
		Syntax: 
			void*malloc(size_t size);
			
		#include <stdio.h>
		#include <stdlib.h>
		int main(){
		int *arr;//arr is pointer
		int n=10;
		arr=(int *)malloc(n * sizeof(int));//allocation memory
		if (arr==NULL){
			printf("memory allocation failed.\n");
			return 1;
		}
		//adding values in arr
		for (int i=0;i<n;i++){
			arr[i]=i+1;
		}
		//printing values
		for (int i=0;i<n;i++){
			printf("%d",arr[i]);
		}
		printf("\n");
		getch();
		return 0;
		}

			
	2. calloc()
		- calloc() stand for contiguous allocation.
		- It allocate memory for an array of n elements,each of size.
		- initilize all memory to zero
		- Return a pointer to allocated memory.
		- if the allocation fails then it return null.
		Syntax: 
			void*calloc(size_t n,size_t size);
			
		#include <stdio.h>
		#include <stdlib.h>
		#include <conio.h>

		void main(){
		int *arr;
		int n=10;
		arr=(int *)calloc(n, sizeof(int));

		if (arr==NULL){
			printf("Memory allocation failed.\n");
			return 1;
		}

		for (int i=0;i<n;i++){
		   printf("%d ",arr[i]);
		}

		getch();
		return 0;
		}

			
	3. realloc()
		- realloc() stand for reallocation.
		- change the size of previously allocated memory block.
		- Can be used to resize memory without losing the contents of original memory.
		- returns a pointer to resize a memory block.
		- If it cannot resize,it may return null and original memory remain valid.
		Syntax:
			void*realloc(void*ptr,size_t size);
			
		#include <stdio.h>
		#include <stdlib.h>
		int main(){
		int *arr;//arr is pointer
		int n=10;
		arr=(int *)malloc(n * sizeof(int));//allocation memory
		if (arr==NULL){
			printf("memory allocation failed.\n");
			return 1;
		}
		//adding values in arr
		for (int i=0;i<n;i++){
			arr[i]=i+1;
		}
		//printing values
		for (int i=0;i<n;i++){
			printf("%d",arr[i]);
		}
		printf("\n");

		n=10;
		arr=(int *)realloc(arr,n * sizeof(int));
		if (arr==NULL){
			printf("memory allocation failed.\n");
			return 1;
		}
		//adding values in arr
		for (int i=0;i<n;i++){
			arr[i]=i+1;
		}
		//printing values
		for (int i=0;i<n;i++){
			printf("%d",arr[i]);
		}

		printf("\n");

		getch();
		return 0;
		}

			
	4. free()
		- frees previously allocated memory.
		- After freeing the memory,the pointer become invalid,so it should be set to null.
		Syntax: 
			void free(void*ptr);

Q.. What is memory leak?
==> A memory leak occur when a program allocates memory dynamically using malloc,calloc,realloc etc but fails to 
	release it free().This causes memory to remain allocated even after it is no longer needed,resulting in 
	wasted memory resources.
	- How to avoid memory leak
		always call free() to deallocate memory that was dynamically allocated.
		#include <stdio.h>
		#include <stdlib.h>
		int main() {
			int *p;
			p = (int*) malloc(5 * sizeof(int));
			if (p == NULL){
				printf("Memory allocation failed\n");
				return 1;
			}
			// use memory
			for(int i = 0; i < 5; i++){
				p[i] = i + 1;
				printf("%d ", p[i]);
			}
			// free memory
			free(p);
			return 0;
		}
		
	2. Dangging pointer:
		A dangling pointer is a pointer that points to a memory location 
		that has already been freed or is no longer valid.
		- dereferencing such pointer leads to undefined behaviour,often causing crashes or unpredictable result.
		- this is dangerous situation because pointer still pointer to memory location that the system might 
			reallocate for another purpose.
			
		=> how to avoid danging pointer.
			- set pointer to null after freeing memory
			
			// free memory
			free(p);
			p = NULL;
